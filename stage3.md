The system supports four main user types:

- Client  
- Full construction companies (from start to turnkey)  
- Partial construction companies (specific phases of the project)  
- Material suppliers (providing building materials)

Front-End (Web Application)

The front-end is the user interface of the system, built using modern web technologies. It allows clients and companies to interact with the platform, including browsing companies, booking services, placing orders, and managing profiles. It communicates with the back-end through HTTP requests and displays data to users.

⚙️ Back-End API

The back-end handles all business logic and processes requests received from the front-end. It manages user authentication, service bookings, project tracking, product orders, and reviews. It also coordinates communication with the database, notification service, and file storage.

🗄️ Database (PostgreSQL)

The database stores all system data, including users, companies, services, bookings, projects, products, orders, invoices, and reviews. It ensures data consistency and supports relationships between different entities in the system.

📧 Notification Service

The notification service is responsible for sending emails to users. It sends booking confirmations, order confirmations, and invoice emails. This component is separated from the main system to maintain clean architecture and scalability.
